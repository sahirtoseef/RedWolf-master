Our Three-Pronged approach

  • Cybersecurity
  • Data Gathering
  • Presentation of Data

We started with an approach to learning about the technologies we would need to learn how to use. We would use resources such as nmap and shodan to run penetration testing.

Penetration Testing on Shodan

Our team spent much time learning about how to find specific threats to Drexel University's systems by looking into which IPs were vulnerable and in what ways.

CVE Details

We then took that information and expanded our approach to learn how to categorize the security threats, and to which devices and IP addresses had which vulnerabilities, mostly utilizing CVE Details.

The data gathering techniques used would create a way for us to easily and efficiently concatenate all the data we had gathered from Pen testing to create a large central database.

Jupyter Notebook

We utilized Jupyter to compile this data by scraping through all relevant website's public records to create one central repository.

NMap functionality

We used nmap functionality in the places where public data could not be reached via Jupyter notebook so that we could compile related details about vulnerabilities for Drexel University.

Finally, we used various technologies to present the large amount of data that was collected from the research and gathering into an easy to digest format so that the client may know how to diagnose its isssues.

Tableau Public

We used our database as the source for all of our visual presentations that we display in the dashboard tab. This allows the user to easily see the most pertinent information and the underlying data easily.

Web Development

This website takes all of that database information and puts it onto this site where it can be easily seen and understood. The last tab looks at all the data holistically so that users can look into specific vulnerabilities.
